# Design: Publish `why` to Homebrew via PyPI + Personal Tap

**Issue:** #82  
**Branch:** `sleipnir/issue-82-homebrew-publish`  
**Status:** Implemented — merged in PR for issue #82

---

## Problem

`why` has no published distribution channel. macOS developers can only install from source. The README currently says "Not yet published to PyPI." This blocks adoption.

## Goal

Enable macOS users to install `why` with:

```sh
brew tap tarungulati1988/why && brew install why
```

And Python users globally via:

```sh
pip install git-why
```

## Non-Goals

- homebrew-core submission (requires star threshold, deferred)
- Linux package managers (deferred)
- Windows distribution (deferred)

---

## Approach: PyPI + Personal Homebrew Tap

Two complementary distribution channels, set up in a single release workflow.

### Channel 1: PyPI (`git-why`)

- Package name: **`git-why`** (avoids `why` namespace collision)  
- CLI entry point stays `why` — `pyproject.toml` `[project.scripts]` is unchanged
- Users install with `pip install git-why` or `uv tool install git-why`
- Published via OIDC Trusted Publisher (no stored API tokens)

### Channel 2: Homebrew personal tap (`tarungulati1988/homebrew-why`)

- New GitHub repo: `tarungulati1988/homebrew-why`
- Formula: `Formula/why.rb` using `Language::Python::Virtualenv`
- Formula `url` points to the GitHub release tarball (auto-generated by GitHub on tag push)
- All Python dependencies declared as `resource` blocks (generated by `homebrew-pypi-poet`)
- `brew audit --formula Formula/why.rb` must pass before the tap is published

## Release Workflow (`.github/workflows/release.yml`)

Triggers on `push` to tags matching `v*.*.*`.

```
v0.1.0 tag pushed
  → run tests (matrix: 3.11, 3.12)
  → build wheel + sdist (uv build)
  → publish to PyPI via OIDC trusted publisher
  → create GitHub Release (auto-generates source tarball)
```

The GitHub Release source tarball URL is deterministic:
```
https://github.com/tarungulati1988/why/archive/refs/tags/v0.1.0.tar.gz
```

This URL is what goes in `Formula/why.rb`.

## Formula Template (`Formula/why.rb`)

The resource blocks below are placeholders — actual SHAs must be computed from the published release tarball using `brew generate-formula-resources` or `homebrew-pypi-poet`.

```ruby
class Why < Formula
  include Language::Python::Virtualenv

  desc "Explains why code is the way it is via git history and LLM synthesis"
  homepage "https://github.com/tarungulati1988/why"
  url "https://github.com/tarungulati1988/why/archive/refs/tags/v0.1.0.tar.gz"
  sha256 "FILL_IN_AFTER_RELEASE"
  license "MIT"
  head "https://github.com/tarungulati1988/why.git", branch: "main"

  depends_on "python@3.12"

  # Resource blocks generated via:
  #   pip install homebrew-pypi-poet
  #   poet git-why
  # Replace these stubs with actual output:
  resource "click" do
    url "https://files.pythonhosted.org/packages/.../click-X.Y.Z.tar.gz"
    sha256 "FILL_IN"
  end

  resource "groq" do
    url "https://files.pythonhosted.org/packages/.../groq-X.Y.Z.tar.gz"
    sha256 "FILL_IN"
  end

  resource "rich" do
    url "https://files.pythonhosted.org/packages/.../rich-X.Y.Z.tar.gz"
    sha256 "FILL_IN"
  end

  resource "tree-sitter" do
    url "https://files.pythonhosted.org/packages/.../tree_sitter-X.Y.Z.tar.gz"
    sha256 "FILL_IN"
  end

  def install
    virtualenv_install_with_resources
  end

  test do
    system bin/"why", "--help"
  end
end
```
